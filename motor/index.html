MCP-native

Your agents can operate it too

Motor exposes its full data and action surface to AI agents over MCP — 33 tools behind the same role gates as the UI. Read-only tools for any valid token; mutating tools for admins only. No separate agent API, no second permission model.

Zero code change

Point your Sentry SDKs at Motor

Motor's ingest speaks the Sentry API — classic /store/ and modern /envelope/. Keep the SDKs you already ship, swap the DSN, and issues arrive grouped by fingerprint. Logs and cron heartbeats ride the same endpoints.

Credential-less by design

No tokens to leak

Motor never stores long-lived GH_TOKEN or GCP credentials in env. The service shells out to your CLI session — every GitHub and GCP action runs as the operator who clicked, with the operator's own permissions. Nothing to rotate, nothing to steal.

# How Motor authenticates to GitHub & GCP
$ gh auth token
$ gcloud auth print-access-token

# → runs as YOUR session, per request
# no GH_TOKEN / GCP_* in env. ever.

Graceful by design

Every surface degrades politely

Start with zero integrations — each unconfigured surface shows a card naming the exact env var that unlocks it, never a crash. And Motor is read-mostly: every destructive action (stop container, purge stream, dispatch CI) asks for explicit confirmation first.
