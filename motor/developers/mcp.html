MCP server & tools

Motor is MCP-native: the same data and actions the dashboard uses are exposed to AI agents over the Model Context Protocol — 33 motor_* tools behind the same Fort role gates as the UI. There is no separate "agent API" to maintain; an agent operating your infra sees exactly what an operator sees.

Two endpoints, one role model

POST /mcp/admin — Fort JWT with role=admin: all 23 read tools + the 10 mutating tools
POST /mcp/user — any valid Fort JWT: the 23 read tools only

Both are Streamable-HTTP MCP endpoints (JSON-RPC over POST; GET/DELETE manage the session). They carry their own Fort auth — a token without the admin role gets 403 from /mcp/admin, and the mutating tools simply do not exist on /mcp/user.

Connecting a client

Any MCP client that supports Streamable HTTP with custom headers can connect. A typical configuration:

{
  "mcpServers": {
    "motor": {
      "type": "http",
      "url": "https://motor.example.com/mcp/user",
      "headers": {
        "Authorization": "Bearer <fort-jwt>"
      }
    }
  }
}

Swap the URL to /mcp/admin (with an admin-role token) when the agent needs to act, not just read. Get the JWT from Fort's login endpoint — see Auth & roles.

The safety model

  • Read vs mutate is structural, not advisory. The 10 mutating tools are registered only on the admin server — a user-endpoint session cannot call them, list them, or discover them.
  • No business logic in the MCP layer. Every tool handler is a thin shim onto the same Deps functions that back the HTTP handlers — the same queries, the same CLI delegation, the same NATS client. An agent can never reach a code path the REST API doesn't have.
  • Typed, server-resolved arguments. Projects and slugs come from typed tool args resolved server-side, never from a spoofable identity field; the underlying queries are parameterised.
  • CLI delegation carries through. motor_dispatch_ci and motor_gcp_projects run as the operator's own gh / gcloud session — an agent inherits exactly the operator's cloud access, and nothing outlives the session.
Destructive tools mean it. motor_nats_delete_stream, motor_nats_delete_consumer, and motor_cloudflare_purge_cache are irreversible. If you automate them, put an approval step in the agent's loop — the role gate authorizes, it does not supervise.
